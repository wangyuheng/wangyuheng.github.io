自定义Github个人首页

Posted on 2020-10-17 In github

筛校招简历时学到的新姿势。现在的年轻人真会玩。

在GitHub上接触一个新项目时，如果有一个详实有效的README，可以极大降低学习成本， 提升用户体验。
上篇文章中也介绍了介绍了如何通过badge等方式，让你的README更直观。

但是另一个很重要的信息总是被忽略，那就是个人介绍。如果把个人也看成一个项目，如何让其他人快速了解你，并且与你合作？无论是获取、还是给予相关帮助。

其实GitHub提供了一个自定义个人首页的方式，只不过隐藏在了隐秘的角落，不为众人所知。

Secret Repository

嘘 This is a secret!

这可是官方告诉我的

整个过程十分简单，只需要新建一个和GitHub 用户名匹配的项目即可。需要满足2个条件

  • 项目必须是Public
  • 选择 Initialize this repository with a README

新建后，即可在个人首页中看到多了一个个人首页窗口。

repository创建后，Github已经用模板预先填充了生成的README文件，按照官方推荐来介绍自己吧

高大上吗？这才刚刚开始

Style

虽然这是一个.md文件，但是支持HTML解析。可玩性是不是瞬间提高了？
但是觉得直男审美，或者技能受限？没关系，我们还可以直接使用前端大佬们最喜欢造的轮子

github-readme-stats

github-readme-stats: 在你的 README 中获取动态生成的 GitHub 统计信息！

可以选择统计指标和自定义样式

或者语言使用比例

也可以把感兴趣或者参与的项目也贴进来

只需要一行引用即可

github-profile-trophy

Add dynamically generated GitHub Trophy on your readme

不喜欢卡片，也可以给自己颁个奖杯

github-profile-views-counter

想统计有多少人访问过你的首页？又是一句话搞定

emoji

千言万语不如一个表情？ 来找找喜欢的Emoji吧

关闭个人首页

有一天自闭了，不想在首页向其他人介绍自己，有以下几种方式

  • 删除README.me文件。
  • 仓库设置为私有。
  • 仓库名称不再与Github用户名匹配。
